pub fn max_idle(&mut self, max_idle: u64)[src]

The maximum number of idle connections the pool can contain at the same time. If a connection goes idle (a query returns) and there are already this number of idle connections in the pool, a connection will be closed immediately. Consider using max_idle_lifetime to close idle connections less aggressively.

  • Defaults to the same value as connection_limit.

pub fn connect_timeout(&mut self, connect_timeout: Duration)[src]

A timeout for acquiring a connection with the check_out method. If not set, the method never times out.

pub fn max_idle_lifetime(&mut self, max_idle_lifetime: Duration)[src]

A time how long an idling connection can be kept in the pool before replaced with a new one. The reconnect happens in the next check_out.

  • Defaults to not set, meaning idle connections are never reconnected.

Panics

Panics if max_idle_lifetime is zero.

pub fn test_on_check_out(&mut self, test_on_check_out: bool)[src]

Perform a health check before returning a connection from the check_out. If the health check fails, a few reconnects are tried before returning the error and dropping the broken connection from the pool.

  • Defaults to false, meaning connections are never tested on check_out.

pub fn health_check_interval(&mut self, health_check_interval: Duration)[src]

Sets the interval how often a connection health will be tested when checking out from the pool. Must be used together with test_on_check_out set to true, otherwise does nothing.

  • Defaults to not set, meaning a test is performed on every check_out.

Panics

Panics if health_check_interval is zero.
